## Dependency specification

1. **Office XML metadata graph** — `ioi-ext:OfficeXmlMetadataFacet` with `dctermsCreated`; linked to `FileFacet` providing `fileName`
2. **$MFT graph** — `ioi-ext:MftFacet` with `created0x10` ($SI) and `created0x30` ($FN); linked to `FileFacet` providing `fileName`

Matching is on case-insensitive filename comparison across two named graphs.

## Invariant predicate φ

```
φ_temp : xsd:dateTime(dctermsCreated) ≤ xsd:dateTime(created0x10)
```

A document's application-written creation time must not exceed the filesystem `$SI` creation timestamp.

## Contradiction condition ¬φ

```
IoI_temp : xsd:dateTime(dctermsCreated) < xsd:dateTime(created0x10)
```

The embedded creation time is **earlier** than the filesystem timestamp — indicating `$SI` was set forward relative to document creation.

## SPARQL signature

```sparql
PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>

SELECT DISTINCT
  ?officeFile ?fileName ?filePath
  ?xmlCreated ?mftSiCreated ?mftFnCreated
WHERE {
  GRAPH <office_case12> {
    ?officeFile a observable:File ;
                core:hasFacet ?xmlFacet, ?officeFileFacet .
    ?xmlFacet a ioi-ext:OfficeXmlMetadataFacet ;
              ioi-ext:dctermsCreated ?xmlCreated .
    ?officeFileFacet a observable:FileFacet ;
                     observable:fileName ?fileName ;
                     observable:filePath ?filePath .
  }
  GRAPH <mft_case12> {
    ?mftFile a observable:File ;
             core:hasFacet ?mftFileFacet, ?mftFacet .
    ?mftFileFacet a observable:FileFacet ;
                  observable:fileName ?mftFileName .
    ?mftFacet a ioi-ext:MftFacet ;
              ioi-ext:created0x10 ?mftSiCreated ;
              ioi-ext:created0x30 ?mftFnCreated .
    FILTER(LCASE(STR(?mftFileName)) = LCASE(STR(?fileName)))
  }
  FILTER(xsd:dateTime(?xmlCreated) < xsd:dateTime(?mftSiCreated))
}
ORDER BY ?filePath
```

## Graph IRI substitution

Replace `<office_case12>` and `<mft_case12>` with your named graph IRIs.

## Implementation note

The Office XML graph contains very few triples (19 in the proof-of-concept) while the MFT graph is large (~18M triples). The cross-graph join resolves efficiently because the Office graph drives the outer loop and the filename filter pushes selectivity into the MFT inner loop.
