Contribute

The IoI Framework is community-driven. Contributions of new anti-forensic cases, IoI rules, and Template Instantiators are welcomed via Pull Request.

What you can contribute

_cases/

Anti-forensic case

A new scenario with ground-truth specification, dependent artifacts, and optionally a paired dataset.

af-NNN.mdcases/data/AF-NNN/
_rules/ + rules/sparql/

IoI rule

A reusable SPARQL signature encoding an invariant predicate and its violation condition.

ioi-NNN.mdioi-NNN.rq
_instantiators/ + instantiators/code/

Template Instantiator

A Python script that maps a specific artifact parser's output to CASE/UCO JSON-LD graphs.

inst-NNN.mdNNN_instantiator.py
templates/

CASE/UCO template

A case-agnostic JSON-LD template defining the facet structure for a new artifact type.

NNN_template.jsonld

Submission process

1

Fork the repository

Fork ioi-framework/ioi-framework ↗ and create a branch named add/af-NNN-short-name or add/ioi-NNN-short-name.

2

Pick the next ID

Check _cases/, _rules/, and _instantiators/ for the highest existing ID and increment. IDs are sequential and never reused.

ls _cases/     # → use AF-013
ls _rules/     # → use IOI-013
3

Add files following the schema

All required front-matter fields must be present. Use standard artifact names. Set status: Community and your GitHub handle as contributor. See CONTRIBUTING.md ↗ for the full schema.

4

Run the local validator

The validator checks required fields, ID format, artifact name vocabulary, and linked file existence. CI also runs this check on every PR — catch errors locally first.

python scripts/validate_frontmatter.py
5

Open a Pull Request

The PR template auto-loads with a checklist. Fill out every section including test results (triplestore, triple count, query time, result confirmation on both baseline and post-manipulation graphs). Assign a reviewer from the maintainer team.

6

Review and merge

CI validates your front matter and checks that all linked files exist. A maintainer or community peer reviewer will review within 14 days. Once approved and merged, the site deploys automatically and your contribution appears on the index pages.

StatusMeaningHow to achieve

Deprecated Superseded, known-broken, or no longer applicable Maintainer decision; original content preserved for reference
