## Overview

The adversary selectively deleted specific URL entries from the Chrome History SQLite database (`Default/History`) while leaving behind corresponding IndexedDB cache directories under `Default/IndexedDB/`. These cache directories contain subdirectories named after visited origins (e.g., `https_www.example.com_0.indexeddb.leveldb`), providing an indirect record of browsing activity that persists even after History deletion.

## Expected invariant

For every domain represented by an IndexedDB cache directory, a corresponding URL entry should exist in the Chrome History database:

```
∀ a₁ ∈ A_Cache, ∃ a₂ ∈ A_History : attr^domain(a₁) = attr^domain(a₂)
```

## Violation predicate

A **Semantic IoI** is raised when:

1. An IndexedDB cache directory for a domain exists in the MFT graph, **and**
2. No corresponding URL entry for that domain is found in the History graph, **and**
3. The USN journal records `DataTruncation`, `DataOverwrite`, or `DataExtend` against the `History` file.

## Artifact dependencies

| Artifact | Parser | Fields used |
|---|---|---|
| Chrome `Default/IndexedDB/` | MFTECmd | `filePath`, `parentPath` |
| Chrome `Default/History` | DB Browser for SQLite | URL `fullValue` |
| `$UsnJrnl:$J` | MFTECmd | `fileName`, `updateReasons` |

## Ground truth

**Baseline**: Both IndexedDB directories and History URL entries are present and consistent.

**Post-manipulation**: Target URL entries removed from `History` SQLite; IndexedDB directories intact; `$UsnJrnl` records `DataTruncation` against the History file.
